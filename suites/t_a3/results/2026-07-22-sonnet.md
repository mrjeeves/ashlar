## Diagnosis: two failure populations, only one is a design signal

Requirement A3 asks whether a reader with no reference, shown a construct,
**states its meaning correctly**. Reviewing every failed bullet against the
snippet it scored shows the rubrics mixed two populations:

1. **Meaning of what is on the page** — what the shown keywords, operators,
   and declarations do. Here the cold readers were strong: spaces, `use`
   visibility, dotted-name layering, `append`/`deep` merging with the right
   composed values, `state` mutability, spread, `??`, `!`, if-expressions,
   route captures, and the three-file composed program were all read
   correctly in substance. **Only 2 of 24 readers made an actively wrong
   claim about shown semantics** (the wrong-claim column above).

2. **System behavior not exhibited in the snippet** — that omitting a merge
   kind on a later layer is a compile error, that stored values are
   validated at startup, that `port` marks the server root, that rejection
   status is 403 specifically, that handlers also serve WebSocket, that
   locals are single-assignment, that there are exactly two runtime faults.
   A reader without the reference cannot state these except by luck;
   roughly 60% of all scored bullets (and the large majority of failed
   ones) are in this population. These bullets measure *reference recall*,
   which a no-reference protocol cannot measure. They are miscalibrated
   rubric, not evidence about the syntax.

The strict number is recorded above and stands; the protocol is the
protocol. But the actionable reading of this run is: **the surface largely
communicates what it shows, and the gate found two real bugs plus a set of
prior-import hazards.**

## Genuine design findings (the gate working as intended)

**F1 — `pipe`/`stack` restatement reads as override.** (`08-handle-pipe`,
wrong-claim.) The reader modeled the derived layer's `handle pipe = ...` as
*replacing* the base handler — "the b.ash version, not file 1's" — where
Ashlar runs every layer in order, each receiving the previous return. The
restated kind on the second layer did not communicate chain-participation;
the reader imported the override prior (which is also Ashlar's own default
for kindless properties, making the guess extra reasonable). This is the
most serious A3 finding: the wrong model produces silently wrong
expectations about which code runs, and no compile error can catch it.

**F2 — `{Shape}` map syntax reads as a set.** (`10-stored`, wrong-claim.)
`stored messages: {chat.data.Message} = {}` was read as "a homogeneous
collection, most naturally a set," explicitly denying the keyed-by-text map
semantics. Braces-around-element reads as set-literal/set-type in the
dominant priors. A key-explicit spelling (e.g. `{text: Shape}`) would make
the map reading unmistakable and is worth a design pass.

**F3 — index-yields-optional is invisible.** (`17-optional-index`, 0/4.)
The reader typed `m["top"]` as plain `number`, never `number?`. Nothing on
the page marks indexing as optional-producing; every mainstream prior
(JS `undefined`, Go zero-value, Python KeyError) disagrees with quiet
optionality. Downstream shape-checking catches misuse at compile time, so
this fails safe (A4), but the reference must carry this rule alone.

**F4 — determinism guarantees contradict priors silently.** Readers
asserted map iteration is unordered (`18-for-map`, Go prior), `every`
durations are unvalidated strings (`21-scheduled`), and foreign shape
checks happen at build time (`23-foreign`). Each is an Ashlar-specific
guarantee the snippet cannot exhibit; each active contradiction shows the
prior that will be imported wherever the reference is not in context.

## Protocol revision for the next run

Recorded in `PROTOCOL.md` (dated): rubric bullets must be decidable from
the snippet text plus universal programming knowledge. Bullets about
behavior the snippet does not exhibit move to a separate, reference-in-
context comprehension suite. Re-baseline after F1/F2 receive a design
decision.
